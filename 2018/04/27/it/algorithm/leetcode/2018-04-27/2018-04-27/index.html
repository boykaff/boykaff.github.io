3. Longest Substring Without Repeating Characters

Posted on 2018-04-27 | In LeetCode

title

Examples:

tags

字符串操作

level

中级

idea

这道题对刚入手练习算法的freshman来说可难可不难，要说简单，当然不用过脑，就O(n^2)时间复杂度内计算出来，但这显然有违初衷，昨儿抽闲暇时间问了几位同事，均暂无好的解法，折腾良久，终于所有case通过了。

声明：maxLen为最大长度，conflictVal重复元素的起始位置，A[]字符串原始数据数组；Arr[]，长为256的整形数组用于存放各元素产生重复的下标值

举个自己弄的一个较有代表性的例子： 字符串（即字符数组）： abcbbcabd，预得结果长度为：4，即”cabd“。
字符数组以及对应的下标：

a b c b b b a c d
0 1 2 3 4 5 6 7 8

申请一个长度为256的整形数组，，并全部初始化为：-1：

… a b c d …
… -1 -1 -1 -1 …

A[0]元素 a，放入数组中，原始值为-1，表明未发生过冲突，直接放入，即：

… a b c d …
… 0 -1 -1 -1 …

依次将下标为1和2的b，c元素下标值放入数组中，即：

… a b c d …
… 0 1 2 -1 …

显然，此时，Arr[a]=0；Arr[b]=1；Arr[c]=2；conflictVal=0；maxLen = 3（“abc”）；
继续放入位置3的元素b，有效长度为2（abcbbcabd，红色部分），对比数组中存在冲突，即将a元素的新下标存入该数组中，即：

… a b c d …
… 0 3 2 -1 …

并将conflictVal值置为1（即该元素产生冲突的起始位置），表明这个是本次冲突的起始下标位置，这个位置之后的数据对其它非重复子字符串的长度统计才有意义。Arr[b]=3；maxLen=3；

继续，A[4]元素b，经过转换，可以将其中的冲突值3替换掉，此时对应的有效长度为1（abcbbcabd），下标3之前的数据不能用作长度计算使用，故将conflictVal置为3，
Arr[b]=4；maxLen=3；

接下来，A[5]的元素c，该元素的冲突下标为2，可以看出只有从下标4开始的abcbbcabd为当前元素对应的最长长度（2），conflictVal的值保持为3，Arr[c]=5；maxLen=3；

A[6] 元素a：abcbbcabd，冲突位置为0，有效长度为3，Arr[a]=6；maxLen=3；

A[7] 元素b：abcbcabd，冲突位置为4，有效长度为3，Arr[b]=4；maxLen=3；

A[8]元素d：abcbcabd，无冲突，有效长度为4，Arr[d]=8，maxLen=4；
